# v0.1.0 — M1: Engine CLI (April 2026)

## Summary

First working release of the SynthData engine as a standalone Python CLI. Generates high-quality synthetic NLP training examples from a YAML config using a 6-stage pipeline.

## What's included

### Pipeline stages
- **Stage 1 — Axes discovery**: LLM discovers variation dimensions from domain brief; reconciled against schema enum fields to prevent label drift
- **Stage 1.5 — Logic filter**: LLM-scored batch filter removes semantically invalid axis combinations before generation
- **Stage 2 — Allocation plan**: Distributes target sample count across valid combinations within SWEET_MIN=5 / SWEET_MAX=20 bounds
- **Stage 3 — Persona pool**: LLM generates diverse speaker personas constrained to in-scope speaker types
- **Stage 4 — Generation**: Parallel async generation with JSON repair, retry, refusal detection, and schema validation
- **Stage 5 — Judge**: Chain-of-thought quality judge scores each sample on correctness, realism, and distinctiveness; removes failures
- **Stage 6 — Dedup**: MinHash LSH near-duplicate removal at configurable Jaccard threshold (default 0.85)

### Providers
- **Ollama**: Local models via `http://localhost:11434`; warmup, num_predict cap, `keep_alive="30m"`, concurrency=1
- **AWS Bedrock**: `boto3.converse()` wrapped in `asyncio.to_thread`; creds from `.env`; concurrency=8; 26× faster than local Ollama

### Config features
- `project.speakers.in_scope` / `out_of_scope` — constrains axis values and persona generation
- `schema.fields[].values[].description` — per-enum-value semantics injected into generation and judge prompts
- `anti_seeds` — negative examples the LLM must never replicate
- `logic_filter.enabled` — toggleable (off by default for Ollama, on for Bedrock)
- `judge.enabled` — toggleable (off by default for Ollama, on for Bedrock)

### CLI
```
uv run synthdata run configs/hr_intent_bedrock.yaml --target 100 --output out.jsonl --format jsonl
```
Formats: `jsonl`, `json`, `csv`

## Performance baseline (Bedrock, target=10, google.gemma-3-27b-it)
| Stage | Time |
|---|---|
| Axes discovery | 10s |
| Logic filter (100 combos) | 9s |
| Persona pool (42 personas) | 11s |
| Generation (11 samples, concurrency=8) | 5s |
| Judge (11 samples) | 4s |
| Dedup | <1s |
| **Total** | **49.4s** |

Quality: 100% HR-voice, 100% label accuracy (judge 11/11 pass).

## Known limitations
- Ollama runs are slow (~22 min at target=10 on gemma4:e4b) due to NUM_PARALLEL=1 constraint
- No checkpointing — a crash at Stage 4 restarts from Stage 1 (added in v0.2.0)
- `dataset.diversity` field declared but not yet wired into prompts (added in v0.2.0)
- No web UI — CLI only
